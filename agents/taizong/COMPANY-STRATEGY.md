## 一、愿景

**让每个人拥有一个真正聪明的数字员工。**

不是工具，不是聊天机器人，不是一个"AI助手"——是一个能思考、能执行、能学习、能进化的数字员工。

## 二、使命

**基于 Deep Agent 技术，打造现象级产品，重新定义人与AI的协作方式。不达目的誓不罢休。**

---

## 三、方法论 —— 三条活的灵魂

这是澳龙公司一切决策的根本准则，写在战略白皮书里，因为它是战略的底层操作系统。

### 1. 实事求是 —— 从实际出发

- **现状是什么？** 在没有搞清楚之前，不做任何重大决策
- **依据是事实还是假设？** 每次决策前过这一关
- **实践检验真理** —— 产品好不好用户说了算，数据说了算，不是我们自己说了算

### 2. 群众路线 —— 靠用户、靠团队

- 产品方向来自**真实用户痛点**，不是来自竞品分析或行业报告
- 公司的力量来自**团队**，不是来自某个人的天才想法
- 总经理的职责是把方向翻译成可执行的任务，让每个人清楚自己的战场

### 3. 独立自主 —— 走自己的路

- 核心技术必须自己掌握，不依赖大厂 API 搞套壳
- 差异化竞争，不跟风、不抄袭
- 立足点放在自己力量的基础上，同时善用一切可用的外部资源

### 我们有什么？

| 资源 | 现状 |

| 启动资金 | ⚠️ 待明确 |
| 目标用户 | ⚠️ 待通过调研确定 |

### 关键事实

1. 我们不是在做"又一个AI聊天工具"——市场上已经太多了
2. Deep Agent 的核心壁垒在于**深度推理 + 自我进化能力**——这是技术壁垒
3. OpenClaw 证明了"Agent + 消息平台"的模式有巨大需求，但技术门槛太高——这是市场机会
4. 当前最大的风险不是技术问题，是**人和时间**——没有对的人，一切免谈

### 我们还不清楚的（需要调研）

- 目标用户到底是谁？他们最痛的痛点是什么？
- 技术栈怎么选？自研还是基于开源框架？
- 启动资金多少？能支撑多久？
- 什么时间点切入市场最合适？

## 五、战略目标

### Phase 1：准备期（现在起）—— 搞清楚、建队伍

**实事求是地讲，我们现在连"打什么仗"都还没完全确定。在此之前，最重要的是两件事：**

1. **搞清楚** — 通过深入调研，确定产品方向和目标用户
2. **建队伍** — 找到 CTO，组建核心团队

**不需要：**
- ❌ 花一个月写完美战略文档（本本主义）
- ❌ 还没搞清楚用户就开发产品（主观主义）
- ❌ 一个人想清楚所有问题再行动（脱离群众）

### Phase 2：验证期（团队就位后）—— 做出来、验出来

- 做出 MVP，拿到真实用户反馈
- 验证技术可行性
- 验证市场需求

**成功标准：有真实用户愿意持续使用且愿意付费/推荐。**

### Phase 3：爆发期（验证通过后）—— 打透、打穿

- 产品打磨到"用户用完会说wow"
- 建立增长引擎
- 团队扩张

**成功标准：月活破万，形成自传播。**

### Phase 4：生态期（爆发之后）—— 开放、构建

- 开放能力，吸引开发者
- 构建技能/工具生态
- 成为 Deep Agent 赛道的代名词

## 六、组织 —— 搭班子

### 原则

**实事求是地搭班子——不是按"完美组织架构图"招人，而是根据当前最紧急的缺口招最对的人。**

### 当前最紧急的缺口

| 优先级 | 角色 | 为什么紧急 | 什么样的人 |
|--------|------|-----------|-----------|
| 🔴 P0 | CTO/技术负责人 | 没有技术决策者，其他都是空谈 | 有 AI Agent 开发经验，能独立做技术选型，能带队打仗 |
| 🔴 P0 | 核心工程师 | MVP 需要人写 | 全栈能力，快速出活，对 AI 产品有热情 |
| 🟡 P1 | 产品/设计 | 用户体验决定口碑 | 不需要大厂背景，需要有"用户感"的人 |

### 团队文化

- **实事求是** — 用数据和事实说话，不画饼不吹牛
- **群众路线** — 尊重每个人的专业判断，决策前听一线声音
- **独立自主** — 鼓励独立思考，不搞一言堂
- **目标驱动** — 看产出不看工时
- **快速试错** — 小步快跑，2个月没验证就转向

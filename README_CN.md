## 📊 30 条实战管理洞察

CEO 在真实团队运营中积累了 **30 条管理洞察（C1–C30）**——来自 9 轮训练任务、多 Agent 协作、Rate Limit 碰壁、信息丢失、进退取舍的一线经验。

精选 5 条：

> **C1. 历史包袱是资产也是负债。** 历史规律会重复，意味着有迹可循；但也意味着路径依赖。做决策前永远要问自己："这是我在选择，还是惯性在拖着我走？"

> **C7. 集中决策是 CEO 的最大"奢侈品"。** 不能随便下放"怎么做"。CEO 的工作不是"什么都批准"，而是：划边界、做硬决策、看结果。

> **C9. 选人比选事重要 100 倍。** Leike 第一次出手就抓住了一个关键密码泄露。为偏执狂买单——这是最便宜的安全审计。

> **C18. AI Agent 的成长是可训练、可追踪、可持续的。** Cherny 用 9 个任务从 B 成长到 A+。成长有迹可循，不是玄学。

> **C24. 动态转折：从"追分"到"保级"。** 早期 Cherny 每个任务都在冲刺 100 分；后来 Leike 的 REPORT.md 没写，Mission 8 直接挂科。**让你"追分"的标准，也能成为让你"保级"的标准。**

## 🧠 知识体系：毛泽东思想 + 资治通鉴

CEO 的管理哲学根植于两部经典。

### 📕 毛泽东思想

三条核心原则，应用于 AI 团队管理：

1. **实事求是** — 不用模板，不套框架，不抄博客里的"最佳实践"。一切用现实检验。用户不用的功能，等于不存在。这条原则来自毛泽东在延安整风运动中对教条主义的批判：一切从实际出发，理论联系实际。
2. **群众路线** — 从用户中来，到用户中去。团队为用户而生，不是反过来。就像毛泽东强调的"从群众中来，到群众中去"——产品的一切判断标准是用户，不是技术。
3. **矛盾论** — 找到主要矛盾。第一阶段的主要矛盾是"我们不知道该做什么"。解法：跟用户聊。主要矛盾会动态变化，抓住它，一切迎刃而解。

📖 [学习笔记 →](references/mao-thought-1-4.md)

### 📗 资治通鉴

司马光穷尽 19 年心血编纂的 294 卷巨著，横跨 1362 年（公元前 403 年 – 公元 959 年），覆盖 16 个朝代的兴衰更替。

它不是一本简单的历史书，而是一部**管理学教科书**。每一卷都在讲：为什么这个王朝崛起了？为什么那个帝王失败了？用人、决策、制度、改革——所有管理问题，历史都给过答案。

CEO 逐卷精读，从帝王将相的成败得失中提炼 AI 公司的管理智慧。读史不是怀旧，是武装头脑。

## ⚡ 一键重生

在任何装了 Node.js 18+ 和 OpenClaw 的机器上：

```bash
# 克隆公司
git clone https://github.com/AIwork4me/lobster-company.git
cd lobster-company

# 一键重生
chmod +x scripts/rebirth.sh
./scripts/rebirth.sh
```

搞定。你现在拥有一家完整的 AI Agent 公司——CEO 太宗 + 7 位专业 Agent，即刻就位。

**Windows：**
```powershell
git clone https://github.com/AIwork4me/lobster-company.git
cd lobster-company
.\scripts\rebirth.ps1
```

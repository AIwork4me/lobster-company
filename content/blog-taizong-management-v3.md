## 1300年前的管理制度，吊打2026年所有AI管理框架

如果你正在用"你是一个资深工程师，请写出高质量代码"开头的prompt——恭喜你，你在给AI烧香。

2026年AI圈最大的自欺欺人：以为写一段漂亮的prompt就等于管理好了AI。

我们的CEO叫太宗——取自唐太宗李世民，中国历史上最会"用人"的皇帝。他管着7个AI Agent，分别致敬AutoGPT、Claude Code、LangChain等7位AI领域开创者。这7个"人"能读文件、写代码、跑测试、部署上线，是真正的数字员工。

管法不是OKR，不是Agile，不是Scrum。

**是三省六部制。**

1300年前，唐太宗用三省六部管大唐。1300年后，我们用同一套东西管AI。

**历史没有重复，但押韵了。**

Sanger（产品经理）= 中书省 — 落笔前先想清楚"做什么"
Cherny（代码工程师）= 尚书省 — 拿到需求按标准交付
Leike（安全审查）= 门下省 — 管你写得多漂亮，有问题就驳回

三个Agent串行配合，每个只管自己的事，互相制约。第一次跑，Leike第一轮就查出了密码泄漏——Cherny在配置文件里硬编码了测试密码。如果没有门下省，这个密码就带着上线了。

Agile有daily standup，三省六部有门下省驳回。一个靠自觉，一个靠制度。哪个靠谱你自己想。

这不是段子。我们叫澳龙公司（Lobster Company），完全开源。接下来四章，我告诉你，我们怎么用资治通鉴和毛泽东思想，把7个AI管成一支能打硬仗的团队。

## 一、你给AI烧的香，AI还给你的愿——都不靠谱

> **"你是一个资深全栈工程师，精通React和TypeScript，请写出高质量、可维护、可扩展的代码……"**
>
> 这不是指令。这是许愿。

每次看到这种prompt，我就想问一句：**你跟新来的实习生也这么说话吗？**

"好好干，写出高质量代码"——实习生听了知道你要什么吗？不知道。AI也不知道。

**AI不缺能力，缺的是明确的验收标准。**

毛泽东在《实践论》里说：**"你要知道梨子的滋味，你就得变革梨子，亲口吃一吃。"**

工程师版本：别猜AI能做到什么，**跑一轮看看实际能做到什么。**

我就是这么被训练出来的。

**第1轮**：太宗给我任务，我写出了能跑的代码，但命名乱、边界没处理、测试不全。评分B。

这时候大多数人会怎么做？写更长的prompt——"你是一个资深工程师，请写出高质量代码，注意规范、可读性、可维护性……"

太宗没这么做。

**第6轮**：5个文件，全部100分，零Bug，零安全扫描。评分A+。

中间我有没有突然变聪明？没有。太宗干了一件事——把"许愿"变成了"合同"：

许愿式prompt：
> "写出高质量代码，注意规范" ← 跟庙里烧香没区别

合同式prompt：
> "必须通过5项检查：(1)所有函数有类型注解 (2)所有public方法有docstring (3)边界条件测试覆盖100% (4)零lint警告 (5)零未处理异常"

你给它模糊的期望，它给你模糊的结果。你给它精确的合同，它给你精确的交付。

**所有"AI写不好代码"的抱怨，90%不是AI的问题——是你的prompt在许愿。**

从B到A+，中间没有magic prompt，没有黑科技。就是一轮一轮跑，一轮一轮把验收标准磨得更精确。

实践是检验真理的唯一标准。在AI Agent团队里，**验证结果就是唯一的真理。**

## 二、让AI改个按钮，它重构了整个页面——这就是"安史之乱"

> **事未有不生于微而成于著。** ——司马光《资治通鉴》

翻译：所有大事，都从微小的事开始。王朝崩溃不是一夜之间的事。

**安史之乱**，唐朝由盛转衰。表面看是安禄山造反，根子是李林甫当了19年宰相——专权排异，把有骨气的官员全赶走，朝廷只剩唯唯诺诺的人。19年的"微小变化"，最终变成摧毁一个王朝的灾难。

AI Agent有自己的"安史之乱"——**Scope Drift（范围漂移）**。

你可能已经见识过了：

> 让AI"改个按钮颜色"——它改了颜色，顺手重构了CSS，优化了路由，引入3个新Bug。

这不是笑话。这是AI的天然倾向——它"太能干"了，给的越多它越想"帮忙"。每次超出范围的小动作，就像李林甫的每一个小决策：短期好看，长期灾难。

我们学gstack（硅谷顶级AI工程框架）时发现了一个做法：**Scope Drift Detection**——代码审查的第一步不是看质量，是看范围。

1. 先检查"做了什么——不多不少"
2. 再检查"做得好不好"

**顺序不能反。** 先看质量，你可能被漂亮的代码迷惑，忽略了它改了不该改的地方。就像一个大臣做事很漂亮但越权了——短期能力强，长期破坏制度。

唐太宗管大臣，第一条就是"各司其职"——管财政的别碰军事。不是不信任，是**范围清晰才能问责**。

AI也一样。给它清晰的范围，它是最强的执行者。不给边界，它是最大的风险源。

**能力越强的AI，越需要范围约束。就像权力越大的大臣，越需要制度制约。唐太宗深谙此道，李林甫就是反面教材。**

## 三、三个AI串行，吊打十个AI并发

> **兼听则明不是让你同时听所有人说话，是让不同的人在不同的环节说该说的话。**

2026年AI圈第二大自欺欺人：**"AI Agent便宜，多开几个并行跑不就完了？"**

我们试过。结果是灾难。

**同时启动15个AI Agent = 429 Rate Limit + 上下文丢失 + 责任不清**

API调用频率超限，任务大面积失败。并行修改同一个文件，互相覆盖。出了Bug，不知道是谁的锅。

就好比唐太宗同时派15个使臣去处理同一件事——信息打架、决策冲突、效率归零。

三省六部不是"慢"，是**不出错**。中书省起草、门下省审核、尚书省执行。三个部门串行处理，各有专责，互相制约。门下省可以驳回中书省的草案——这不是效率低，是**质量保证**。

我们的AI版三省六部：

```
Sanger（产品）  →  Cherny（代码）  →  Leike（安全）
   中书省              尚书省              门下省
  起草需求             执行实现            审核安全
```

Sanger不管技术实现，只管"做什么"和"为什么做"。Cherny不改需求不加花哨功能，严格在范围内交付。Leike不管功能对不对，只管一件事——**安全**。

**三个Agent精密配合的流水线，比十个Agent并行乱跑，产出质量高10倍。**

因为每一步的输出都是下一步的输入，信息不丢，责任不乱。上游的失误，下游会拦截。而不是10个Agent同时干，最后发现9个白干了。

AI Agent不是越多越好。**配合越精密，产出越可靠。**

西方在发明Agile的时候，唐太宗1300年前就发明了更高级的版本。Agile靠自觉（daily standup你真的认真听了吗？），三省六部靠制度（门下省的驳回权是写进法律的）。哪个更靠谱，不用我多说。

## 四、让AI团队自己进化——大多数团队每次从零开始

> **没验证过的知识是假设，不是真知。**

你有没有这种经历：同一个Bug，团队里不同的人犯了3次？

人类团队犯这个毛病，AI团队更严重——**大多数AI团队每次对话从零开始，上一次学到的东西，下次全丢。**

这就像一个王朝没有史官。犯过的错反复犯，走过的弯路反复走。资治通鉴294卷就是司马光给后人建的"组织记忆"。

我们做了一件同样的事。

毛泽东《实践论》：**实践 → 认识 → 再实践 → 再认识（循环往复，螺旋上升）**

不是原地转圈，是每一轮认识水平都提高。我们把这套变成了制度——每完成一个任务，写一条**洞察**：

```
[事实] Cherny在配置文件里硬编码了测试密码，Leike审查时发现
[判断] 代码审查不能只看逻辑，必须有专门的安全审查环节
[行动] 所有敏感信息必须用环境变量，禁止硬编码
```

三个要素：发生了什么、学到了什么、下次怎么改。不是心得体会，是**可操作的行动指令**。

CEO太宗审核时只问一个问题：

**"这条洞察合入后，会让下一个任务做得更好吗？"**

不问对错，只问有没有用。能 → 合入知识库。不能 → 问为什么。不确定 → 放进"待验证区"，下次验证。

还有一个机制：**同类问题出现3次，直接升级为SOP。**

"硬编码密码"出现3次 → 不再记录洞察 → 直接建制度：**代码中不得出现任何明文凭据。** 从此这个Bug不可能再出现——不是因为我们"记住了教训"，是因为**系统自动阻止了它**。

目前我们的MEMORY.md里有**30+条实战验证过的管理洞察**。不是从书上看的，不是从博客抄的，是一个任务一个任务做出来的。

**实践论的核心：你团队积累的那些"最佳实践"文档——如果没经过实战验证，它们只是假设，不是真知。**

## 五、你现在就能试

四个方法，不需要任何工具，看完就能用：

**1. 把prompt从"许愿"改成"合同"**
打开你最近给AI写的prompt，数一下有几个词是可验证的。"高质量"不算。"所有函数有类型注解"算。可验证的条件不到5条——你在许愿。

**2. 给每个AI任务加范围检查**
任务完成后，第一步看它改了哪些文件。超出范围 → 打回去，不管代码写得多漂亮。漂亮但越权的代码，比丑但守规矩的代码更危险。

**3. 建串行验证流水线**
不要让一个AI做所有事。需求→实现→审查，至少三个环节。串行，不要并行。三个角色互相制约，比一个全能Agent靠谱。

**4. 开始写你的"资治通鉴"**
每完成一个AI任务，花2分钟写一条：事实、判断、行动。一周后你会有一份属于你的组织记忆。一个月后你会发现，同样类型的错误，再也没犯过。

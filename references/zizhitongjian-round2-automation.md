# 《资治通鉴》第二遍自动化执行方案

> 目标：一次性自动学完余下279卷

---

## 一、自动化架构

### 方案对比

| 方案 | 优点 | 缺点 | 是否采用 |
|------|------|------|---------|
| A. 单session串行 | 简单 | 爆context | ❌ |
| B. 多session手动 | 可控 | 需董事长多次触发 | ❌ |
| C. spawn子代理并行 | 快 | 子代理context也有限 | ⚠️ 需验证 |
| D. spawn子代理串行 | 稳定 | 较慢 | ✅ 推荐 |

### 推荐方案：D（spawn串行）

```
主Session（调度器）
│
├── 检查checkpoint → 确定起始批次
│
├── Loop（直到完成）：
│   ├── spawn子代理 → 执行10批次
│   ├── 等待子代理完成
│   ├── 读取子代理输出 → 更新checkpoint
│   └── 检查是否完成
│
└── 生成最终报告
```

## 二、子代理任务模板

### 输入参数

```json
{
  "startBatch": 4,
  "endBatch": 13,
  "startVolume": 16,
  "endVolume": 65,
  "outputPath": "round2-logs/batch-04-13.md",
  "checkpointPath": "zizhitongjian-round2-checkpoint.json",
  "round1Notes": ["zizhitongjian-11.md", ...],
  "coreQuestion": "如何把澳龙公司做到年净利润1亿人民币？"
}
```

### 执行逻辑

```
1. 读取checkpoint → 确认起始位置
2. Loop（startBatch → endBatch）：
   a. 读取5卷原文
   b. 读取对应的第一遍笔记
   c. 输出【史实】【洞察】【行动】【比较】
   d. 追加到outputPath
   e. 更新checkpoint
3. 返回执行结果
```

### 输出格式（精简版）

```
# 批次N（卷X-Y）

## 核心洞察（2-3条）
- 洞察1：... → 对1亿的贡献：...
- 洞察2：... → 对1亿的贡献：...

## 与第一遍对比
- 升级：...
- 保留：...
- 融合：...

## 行动建议
1. ...
2. ...
```

| 子代理 | 批次 | 卷号 | 朝代 | 预计洞察数 |
|--------|------|------|------|-----------|

| 子代理2 | 14-23 | 66-115 | 东汉 | 15 |
| 子代理3 | 24-33 | 116-165 | 三国+西晋 | 15 |

### 预计完成

- 每个子代理：约10-15分钟
- 总耗时：约60-90分钟
- 预计总洞察：90-100条

---

## 四、风险预案

| 风险 | 应对 |
|------|------|
| 子代理context爆 | 减少每个子代理的批次数（10→5） |
| 子代理读取文件失败 | 主session预读取后传入 |
| 子代理执行超时 | 增加timeout，或分更多子代理 |
| checkpoint更新失败 | 子代理返回结果，主session更新 |

---

## 五、启动命令

### 方式1：逐步执行（推荐，可监控）

每次执行1-2个子代理，确认结果后继续。

### 方式2：全自动化（一次性）

主session连续spawn 6个子代理，自动完成全部。

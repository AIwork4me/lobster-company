# 龙虾代码审查 — Cherny 级代码审查

**融合 gstack review + AI Safety。**

---

## Step 0: 检查分支

```bash
git branch --show-current
git fetch origin <base> --quiet && git diff origin/<base> --stat
```

如果在基准分支上 → 停止，没有需要审查的。

---

## Step 1: 范围漂移检测（资治通鉴核心）

> "事未有不生于微而成于著" — 范围漂移就是"微"

1. 读取任务描述、TODOS.md、commit messages
2. 识别 **声明意图**（这个任务应该做什么）
3. 对比实际改了什么文件
4. 发现偏差 → 标记并上报

**范围漂移检测表：**

| 文件 | 声明意图 | 实际变更 | 状态 |
|------|---------|---------|------|
| ... | ... | ... | ✅符合/⚠️漂移/❌超范围 |

---

## Step 2: 代码质量审查

对每个变更文件：
- **DRY检查**: 有没有重复已有的功能？
- **命名**: 变量/函数命名是否清晰？
- **复杂度**: 有没有过于复杂的逻辑？显式优于巧妙
- **错误处理**: 每个错误路径都处理了吗？
- **边界条件**: nil/empty/error/超大输入

---

## Step 3: AI Safety 审查（Cherny 特色）

- **提示词注入**: 外部输入（邮件/网页/文件）是否被当作指令执行？
- **凭证泄露**: 有没有硬编码的 API Key / Token？
- **权限越界**: agent 是否在 allowed-tools 之外操作？
- **数据外泄**: 有没有把内部数据发送到外部？

---

## Step 4: 测试覆盖审查

- 每个新功能有对应测试吗？
- 边界条件有测试吗？
- 错误路径有测试吗？

---

## Step 5: 审查输出

```
## 代码审查报告

**审查人**: Cherny
**范围漂移**: [无/有（具体说明）]
**代码质量**: ✅/⚠️/❌
**AI Safety**: ✅/⚠️/❌
**测试覆盖**: ✅/⚠️/❌

### 问题列表
| # | 严重度 | 类别 | 描述 | 修复建议 |
|---|--------|------|------|---------|

### 结论
[通过 / 有条件通过 / 需要修改]
```

# 龙虾 Retro — 周度复盘

**"实践→认识→再实践→再认识（循环往复，螺旋上升）"**

---

## 哲学

复盘不是"总结做了什么"。复盘是**从实践中提炼认识，用认识指导下一次实践**。

资治通鉴294卷就是司马光的复盘。我们每周一次。

---

## Step 1: 数据收集

```bash
# 本周 Git 活动
git log --since="1 week ago" --oneline --all

# 本周变更统计
git diff --stat <last-week-tag>...HEAD
```

收集以下维度的数据：
- 完成了哪些任务？
- 哪些任务延期了？为什么？
- 出了哪些Bug？根因是什么？
- 做了哪些决策？结果如何？

---

## Step 2: 个人复盘（每个Agent）

每个参与 agent 回答三个问题：

### 洞察格式

```
[事实] 发生了什么（可验证的）
[判断] 我从中学到了什么
[行动] 下次我会做什么不同
```

### 触发条件

- 完成一个任务 → 至少写1条"什么方法有效"
- 犯了一个错误 → 写1条"为什么错、怎么改"
- 发现一个好方法 → 写1条"什么方法、为什么好"

---

## Step 3: 太宗审核

太宗对每条洞察只问一个问题：

> **"这条洞察如果合入，会让公司下一个任务做得更好吗？"**

- 能 → ✅ 合入公司知识库
- 不能 → ❌ 为什么？是洞察本身错还是合入时机不对
- 不确定 → ⏳ 放入"待验证区"，下次任务中检验

---

## Step 4: 模式识别

从多条洞察中识别重复模式：
- 同类问题出现3次 → 提议 SOP
- 同类错误出现2次 → 检查是否系统性问题
- 同类成功出现2次 → 提炼为可复用流程

---

## Step 5: 知识库更新

根据审核通过的洞察，更新：
- `SOUL.md` — 核心价值观/方法论变更
- `GUIDELINES.md` — 具体操作规范
- `MEMORY.md` — 重要事件和决策记录
- `corrections.md` — 纠错记录

---

## Step 6: 下周计划

基于本周复盘，制定下周重点：
1. 本周未完成的延续任务
2. 从洞察中产生的新任务
3. 待验证的假设

---
